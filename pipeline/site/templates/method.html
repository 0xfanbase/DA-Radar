Method & Audit

How this site works

For each jurisdiction in the registry below, a daily watcher polls that jurisdiction's own official regulator feeds for new publications, filters them for digital-asset relevance, and queues the relevant ones. An ANALYST pass drafts a summary card for each queued item with cited sources. A separate VERIFIER pass, in a fresh context with no access to the analyst's reasoning, re-fetches every citation and checks it against the source text, stripping or flagging anything unsupported. A deterministic, non-bypassable gate then re-checks every citation itself before anything is published — it never trusts either pass's self-report. The same watcher/analyst/verifier/gate pipeline code runs unchanged for every jurisdiction; only each jurisdiction's own config file (regulator list, feed URLs, pillar keywords) differs.

A unified pillar taxonomy, applied honestly

Every jurisdiction on this site is described using the same set of regulatory pillars, rather than each jurisdiction inventing its own categories — that consistency is what makes cross-jurisdiction comparison on this site possible at all. This taxonomy was originally drawn up around Hong Kong's own regime, the founding jurisdiction, and is meant to generalize, not to quietly assume every other jurisdiction's regulatory structure maps onto it perfectly. Where a jurisdiction genuinely has no dedicated regulatory regime for a given pillar, its pillar card says exactly that (the "No dedicated regime" seal) rather than being left off the board or filled in with a guess. If a future jurisdiction's regime turns out not to fit this taxonomy at all, that is treated as a reason to revisit the taxonomy, not to force-fit the jurisdiction.

Corrections log

Corrections are always human-initiated and reviewed via pull request before publishing — this project deliberately does not let the audit loop (or anything else) auto-publish a correction on its own, since getting a correction itself wrong would be a more serious integrity failure than the original error. Every correction's own new citation is re-checked by the same deterministic verification gate as any other card change before the pull request can merge.

Editorial rules

Primary sources only for facts. Quotes are capped at 15 words, one per source. Named entities (licensees, applicants, enforcement targets) are mentioned only as primary sources state them, with no commentary. No predictions beyond officially stated timelines. These rules apply identically in every jurisdiction on this site.
