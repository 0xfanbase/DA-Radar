Method & Audit

How this site works

A daily watcher polls official SFC and HKMA RSS feeds for new publications, filters them for digital-asset relevance, and queues the relevant ones. An ANALYST pass drafts a summary card for each queued item with cited sources. A separate VERIFIER pass, in a fresh context with no access to the analyst's reasoning, re-fetches every citation and checks it against the source text, stripping or flagging anything unsupported. A deterministic, non-bypassable gate then re-checks every citation itself before anything is published — it never trusts either pass's self-report.

Corrections log

No corrections have been issued yet. When a published card is later found to need correction, it will be listed here with the date and reason. Corrections are always human-initiated and reviewed via pull request before publishing — this project deliberately does not let the audit loop (or anything else) auto-publish a correction on its own, since getting a correction itself wrong would be a more serious integrity failure than the original error.

Editorial rules

Primary sources only for facts. Quotes are capped at 15 words, one per source. Named entities (licensees, applicants, enforcement targets) are mentioned only as primary sources state them, with no commentary. No predictions beyond officially stated timelines.
